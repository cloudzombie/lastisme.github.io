Last is me! Faq


How does this work?
"Last is me" is a lottery where the winner is the last player buying a ticket! Every square in the home page is a lottery with a duration and a ticket price as the title. If no one buy a ticket throughout the lottery duration, the last one that did it gets the jackpot! As easy as that!
In the square you can see also the Jackpot amount in dollars, an approximate countdown when the lottery expire, and the actual winner. The countdown is resetted if someone buy a ticket before the duration expires. The last winner is also updated accordingly.

Are you running away with the jackpot?
Every lottery is a smart contract written on the Ethereum platform. Execution is granted by the network and the running program is verifiable and unchangeable. In the lottery detail page there is a link to a third party site (example) verifying the deployed code does exactly what we explain in this pages.

How to buy a ticket?
To buy a ticket you only need an Ether Wallet such as Mist.
If you have a browser enabled or an eth client on you machine with rpc port open at 8545 and account 0 unlocked (example), you can buy a ticket by simply pressing a button, however this is not required and if you feel more confortable you can just copy and paste the contract address (you can find it in the contract details) on your (light) wallet with the right amount and send the transaction.

How the countdown is calculated?
The countdown is an approximation of the lottery expiration. This is because under the hood the lottery duration is measured in number of blocks in the ethereum blockchain. This blocks are expected to be dicovered with an average time but they can occur with variance. For example even if the average time between blocks is 14 seconds in the last 1000 blocks, the next block can be found in the next second or in a couple minutes. The countdown is therefore calculated as the remaining blocks to lottery expiration multplicated by the average block time. The greater the number of remaining blocks, the more accurate is the approximation, viceversa the approximation could be misleading cause the mentioned variance.

Why using number of blocks instead of block timestamp as duration?
The timestamp is a little tweakable by miners, to avoid this possible manipulation we decided to refer to the number of blocks.

I won but I didn't receive anything!
Since the winner is found only when the lottery duration is expired, the jackpot is sent only when the first player of the new cycle buy a ticket. If you are the winner and you are in hurry you can buy the ticket yourself.

I accidentally send more ether than required to buy a ticket.
Don't worry, the amount over the ticket price is sent back to you.

What are the strings at the end of the page?
They are the address of the server providing information for the lotteries. If you have a localhost provider running on port 8545, synced to the last block, it will be used. Otherwise it will be used an ethereum public node. The public node is used for your convenience but the best solution for privacy and speed is to use your own node.

I don't have an ether wallet, which one are ok for this service?
We reccomend Mist as client wallet. If you don't want to download the blockchain on your pc we tested Holytransaction multi currency wallet and it works (but you are not in control of the private keys). We tested Jaxx and it cannot make a good estimation of the gas price at the time of writing. We tested MyChromeEtherWallet and it works only manually specifying the gasprice limit. If you want to suggest other wallet add an issue on github.

I am manually forging the transaction which is the reccomended gas limit?
We reccomend to use 200k as gaslimit, however it usually requires much less. The first transaction starting a new cycle in the lottery is the more expensive, about 150k. Usually it requires around 50k.

When I buy a ticket it takes a while to update
The udate of the lottery depends on how Ethereum miners find blocks. Usually they takes 14 seconds but it depends and it is not upon us.

Could not unlock signer account
This message means that you are trying to buy the ticket automatically but you haven't unlocked account 0. See "How to launch geth?"

How to launch geth?

geth --rpc --rpcaddr="localhost" --rpcport="8545" --rpccorsdomain="http://lastis.me" --unlock=0

You can launch geth with account 0 unlocked to buy ticket just by pressing a button. If you are not confortable, unlocking the account is absolutely not mandatory and you can buy tickets copy & pasting the contract address from the lottery detail page to your wallet. Neither having a working client on localhost is mandatory, however it's reccomended because it's faster.

How to create the transaction from console? (using geth for example)
For example for contract 0x2f47E750205249D3240E1E5be41b0069eEEF1067

eth.sendTransaction({from: eth.accounts[0], to: '0x2f47E750205249D3240E1E5be41b0069eEEF1067', value: web3.toWei(10, 'finney'), gas:200000, gasPrice:20000000000 });


Fees
The fee of the house is just 1% actually. The lottery owner could change this but no more than 2% will ever be possible as you can see from the contract code. All the rest amount, at least 98%, will partecipate to the jackpot, at least 90% to the same lottery, at most 8% become jackpot of other "Last is me" lottery.
